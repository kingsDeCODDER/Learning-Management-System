Python File Handling

Introduction to File Handling

File handling is an essential concept in Python, allowing you to read, write, and manipulate files stored on your computer. Python provides several built-in functions like open(), read(), write(), and close() to manage files.

The most common file modes are:

  • 'r': Read mode (default). Opens the file for reading.
  • 'w': Write mode. Creates a new file or overwrites an existing file.
  • 'a': Append mode. Opens the file and appends new data to the end.
  • 'x': Exclusive creation mode. Creates a new file and raises an error if it already exists.

Opening and Reading Files

To read a file in Python, you use the open() function in read mode. You can read the entire content of the file, or you can read it line by line. 

# Open the file in read mode
file = open("example.txt", "r")

# Read the entire content of the file
content = file.read()
print(content)

# Always close the file after reading
file.close()

You can also read the file line by line using a loop:

# Read file line by line
file = open("example.txt", "r")

for line in file:
    print(line.strip())

file.close()

Writing to Files

Writing to a file is just as simple. You use the open() function with the write mode 'w' or append mode 'a'. The 'w' mode overwrites the file, while the 'a' mode appends new data to the end of the file. 

# Open the file in write mode ('w')
file = open("example.txt", "w")

# Write some content to the file
file.write("Hello, world!\n")
file.write("This is a new file.")

# Close the file after writing
file.close()

If you want to add more content without overwriting the existing content: 

# Open the file in append mode ('a')
file = open("example.txt", "a")

# Append new data to the file
file.write("\nThis text is appended.")

file.close()

Working with Context Managers

A more efficient way to handle files is by using a context manager, which ensures the file is closed automatically after the block of code is executed. This is done using the with statement. 

# Using a context manager to read a file
with open("example.txt", "r") as file:
    content = file.read()
    print(content)

# No need to manually close the file

This approach is safer and cleaner because it ensures the file is always properly closed.

Handling File Exceptions

When working with files, it's important to handle errors, such as when a file does not exist. You can use try and except blocks to manage these situations. 

# Try to open a file that may not exist
try:
    with open("non_existent_file.txt", "r") as file:
        content = file.read()
except FileNotFoundError:
    print("The file was not found!")
