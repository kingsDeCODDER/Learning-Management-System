Object-Oriented Programming in Python

What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that uses "objects" to model real-world things. It allows for better structure and modularity by organizing code into classes and objects.

Classes and Objects

A class is a blueprint for creating objects. Objects are instances of classes. 

class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    def info(self):
        return f'{self.year} {self.make} {self.model}'

In the example above, Car is the class, and it contains attributes (make, model, year) and methods like info().

Creating an Object

Once you have a class, you can create objects from it:

my_car = Car('Toyota', 'Corolla', 2020)
print(my_car.info())

This creates an object my_car of the Car class, and you can access its methods and attributes.

Inheritance

Inheritance allows a class to inherit attributes and methods from another class. This promotes code reuse. 

class ElectricCar(Car):
    def __init__(self, make, model, year, battery_size):
        super().__init__(make, model, year)
        self.battery_size = battery_size

    def battery_info(self):
        return f'{self.battery_size}-kWh battery'

The ElectricCar class inherits from the Car class but adds additional functionality.

Encapsulation and Polymorphism

Encapsulation refers to bundling data (attributes) and methods (functions) that operate on the data into a single unit, or class.

Polymorphism allows different classes to use the same method names, even if their behavior is different. 

class Animal:
    def speak(self):
        return "Sound"

class Dog(Animal):
    def speak(self):
        return "Bark"

class Cat(Animal):
    def speak(self):
        return "Meow"

animals = [Dog(), Cat()]
for animal in animals:
    print(animal.speak())

In this example, both Dog and Cat inherit from Animal, but their speak() methods behave differently.
