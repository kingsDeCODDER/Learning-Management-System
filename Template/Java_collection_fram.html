Java Collections Framework

What is the Collections Framework?

The Java Collections Framework provides a set of classes and interfaces to store and manipulate groups of objects as a single unit, such as lists, sets, maps, and queues. It makes the task of working with data structures easier and more efficient.

Main Interfaces in the Collections Framework

The core interfaces in the Java Collections Framework are:

  • Collection - The root interface for all collection types.
  • List - An ordered collection (e.g., ArrayList, LinkedList).
  • Set - A collection that contains no duplicate elements (e.g., HashSet, TreeSet).
  • Queue - A collection designed for holding elements prior to processing (e.g., PriorityQueue, LinkedList).
  • Map - A collection of key-value pairs (e.g., HashMap, TreeMap).

List Example

A List in Java maintains the order of insertion and allows duplicate elements. The most commonly used implementations are ArrayList and LinkedList. Here's an example using ArrayList:

import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Orange");

        System.out.println("Fruits: " + list);  // Outputs: Fruits: [Apple, Banana, Orange]

        list.remove("Banana");
        System.out.println("After removal: " + list);  // Outputs: After removal: [Apple, Orange]
    }
}

Set Example

A Set is a collection that contains no duplicate elements. The most common implementations are HashSet and TreeSet. Here's an example using HashSet:

import java.util.HashSet;

public class Main {
    public static void main(String[] args) {
        HashSet<Integer> set = new HashSet<>();
        set.add(1);
        set.add(2);
        set.add(3);
        set.add(2);  // Duplicate element

        System.out.println("Set: " + set);  // Outputs: Set: [1, 2, 3] (duplicates are ignored)
    }
}

Queue Example

A Queue is used to hold elements prior to processing and typically follows a FIFO (First In, First Out) order. Here's an example using PriorityQueue:

import java.util.PriorityQueue;

public class Main {
    public static void main(String[] args) {
        PriorityQueue<Integer> queue = new PriorityQueue<>();
        queue.add(5);
        queue.add(10);
        queue.add(1);

        System.out.println("Head of the queue: " + queue.peek());  // Outputs: Head of the queue: 1

        queue.poll();  // Remove the head element
        System.out.println("After removing head: " + queue);  // Outputs: After removing head: [5, 10]
    }
}

Map Example

A Map is a collection that maps keys to values. It does not allow duplicate keys. The most common implementations are HashMap and TreeMap. Here's an example using HashMap:

import java.util.HashMap;

public class Main {
    public static void main(String[] args) {
        HashMap<String, Integer> map = new HashMap<>();
        map.put("Apple", 10);
        map.put("Banana", 20);
        map.put("Orange", 30);

        System.out.println("Map: " + map);  // Outputs: Map: {Apple=10, Banana=20, Orange=30}

        map.remove("Banana");
        System.out.println("After removal: " + map);  // Outputs: After removal: {Apple=10, Orange=30}
    }
}

Iterating Over a Collection

You can use an Iterator or a for-each loop to traverse through elements in a collection. Here's an example of iterating through a List using a for-each loop:

import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Orange");

        for (String fruit : list) {
            System.out.println(fruit);
        }
    }
}

Advantages of the Collections Framework

  • Reusable code: The framework provides standard interfaces and classes for data structures that can be reused in different applications.
  • Performance improvements: Algorithms and data structures are optimized for better performance.
  • Thread safety: Some implementations in the framework, like ConcurrentHashMap, are thread-safe.
  • Flexibility: Collections can grow dynamically, making them more flexible than arrays.
