Python Exception Handling

What is Exception Handling?

Exception handling in Python allows you to manage and respond to errors or exceptions that occur during program execution. It ensures that the program doesn't crash and allows for graceful error handling.

Try and Except Block

The basic structure for exception handling in Python is the try and except block. 

try:
    x = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")

In this example, the try block will raise a ZeroDivisionError, which is then caught by the except block, preventing the program from crashing.

Catching Specific Exceptions

You can catch specific exceptions to handle them in different ways: 

try:
    number = int(input("Enter a number: "))
except ValueError:
    print("That's not a valid number!")

If a user enters something that cannot be converted into an integer, the ValueError exception will be raised, and the message will be displayed.

Finally Block

The finally block is executed no matter what, whether an exception is raised or not. It is typically used for cleanup actions. 

try:
    file = open("data.txt", "r")
except FileNotFoundError:
    print("File not found!")
finally:
    print("Closing file operation.")

Even if the file is not found, the finally block will still execute.

Else Block

The else block runs if no exceptions are raised in the try block. 

try:
    x = 10 / 2
except ZeroDivisionError:
    print("Division by zero!")
else:
    print("Division successful!")

Here, if the division is successful, the else block will run.
